# commit: 規約に沿ったコミット

ローカルの変更を、表示幅ルールを含むコミットメッセージ規約に従ってコミットする。

このスキルはコミット作業一式を **サブエージェントへ委譲** し、メインコンテキストには
結果サマリだけを残す。

## 手順（メインエージェント）

1. **自分で git の調査・ステージ・コミットを行わない。** `Task` ツールでサブエージェント
   （`general-purpose`）を1つ起動し、コミット作業一式を委譲する。

2. サブエージェントへのプロンプトには次を含める。

   - `~/.claude/skills/commit/references/commit-procedure.md` を読み、その手順に厳密に
     従ってローカルの変更をコミットする旨。
   - ユーザーがコミット対象・粒度・メッセージに関する指示を出していれば、その内容を
     そのまま伝える（例:「認証まわりの変更だけコミット」「2コミットに分けて」）。
   - 戻り値は簡潔なサマリのみとする旨。手順書の「呼び出し元へ返すサマリ」に従わせる。

3. サブエージェントが返したサマリをユーザーに提示する。エラーや「コミット対象なし」
   が返った場合も、その内容をそのまま伝える。

判断・実行はすべてサブエージェント側で完結させる。メインエージェント自身が git を叩いたり
規約・diff を読み込んだりしないこと。
