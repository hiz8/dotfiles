# ブランチ作成

タスク内容からブランチ名を自動生成、または引数で指定されたブランチ名を使って新しいブランチを作成・切り替える。

## 手順

1. 以下のコマンドを並列実行して現在の状態を把握する:
   - `git status` で未コミットの変更を確認（`-uall` フラグは使わない）
   - `git branch --show-current` で現在のブランチ名を取得

2. 未コミットの変更（ステージ済み・未ステージ・未追跡）がある場合は、ユーザーに警告して確認を取る。続行の許可がなければ**停止**する。

3. ブランチ名を決定する:
   - 引数 `$ARGUMENTS` が指定されている場合はそれをブランチ名として使用
   - 未指定の場合はタスク内容や会話のコンテキストからブランチ名を自動生成する
   - 命名規則: `type/description`（例: `feat/add-theater-info`, `fix/screening-time-parse`）
   - type の種類: `feat`, `fix`, `chore`, `ci`, `docs`, `refactor`, `test` など
   - description は英語・ケバブケースで簡潔に

4. 同名のブランチが既に存在しないか確認する:
   - `git branch --list <branch-name>` でローカルを確認
   - 既に存在する場合はユーザーに通知して**停止**する

5. `git checkout -b <branch-name>` でブランチを作成・切り替える。

6. 作成されたブランチ名をユーザーに出力する。

## 重要なルール

- 既存ブランチと同名のブランチは作成しない
- ブランチ名は英語・ケバブケースで `type/description` 形式
- `main` や `master` というブランチ名は作成しない
- 引数が `type/description` 形式でない場合は、適切な type を推測して付与する
